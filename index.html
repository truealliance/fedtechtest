Introduction

Here is a quick coding challenge. It is important to note that this is by no means a test. We just want to get a sense of how you write code and solve problems.

Imagine we are collaborating on this webpage and there are 4 tasks assigned to you:

  1. Fix the layout issue.

  2. Add new features to the gallery:

    1. Add lazy load.
    2. Use CSS grid.
    3. Add a "Load more" button.

Please follow the guide in each section and try to complete the tasks. Feel free to modify the code.

Task 1: Overall Layout

As you can see, the footer section is not sitting at the bottom of the window if the viewport is very tall (e.g. set the viewport to 1400x2000). Can you please try to make it always sits at the bottom?

Note: Don't use position: fixed; because we don't want to overlap the page content.

Task 2: Gallery

Here is a flexbox gallery with dummy images from picsum.photos. We would like to add a few improvements:

  1. Task 2.1 Lazy loading: We want to improve the performance with lazy loading. Bonus if you could also improve CLS.

  2. Task 2.2 Convert to CSS grid: Can you please add a .gallery--grid modifier to make a masonry layout similar to the image below? You can follow the Picsum's guide to change the image ratios (e.g. https://picsum.photos/id/1001/200/300).

  3. Task 2.3 Load more: Can you implement a "Load more" button? Clicking on it will load another 12 images from picsum.photos and append below the current gallery.
